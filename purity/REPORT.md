# GORM Purity Survey Results

This document summarizes the purity behavior of `*gorm.DB` methods across all surveyed GORM versions.

## Legend

### Purity
- ✅ **Pure**: Method does NOT pollute the receiver
- ⚠️ **Impure-Overwrite**: Pollutes receiver, but repeated calls overwrite (less dangerous)
- ☠️ **Impure-Accumulate**: Pollutes receiver, repeated calls stack up (DANGEROUS)

### Immutable-Return
- ✅ **Immutable**: Returned `*gorm.DB` can be safely reused/branched
- ☠️ **Mutable**: Returned `*gorm.DB` is mutable (branches interfere)

### Clone Values
- `0`: No cloning (DANGEROUS - mutations leak)
- `1`: Clone Statement with empty Clauses
- `2`: Full clone (Statement.clone(), keeps Clauses)
- `-1`: Not detected / N/A
